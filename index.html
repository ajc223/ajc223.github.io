Method 1 - PJAX

Plugins like Swup (https://github.com/gmrchk/swup) use pjax to replace content & speed up page loads. It also adds classes dynamically for state changes (is animating, is leaving, etc) which give greater flexibility for css transitions.

The issue is that these types of plugins do not reload the entire page, so any other javascript is not processed. There are ways around it which require including all JS in a single file and conditionally determining what runs when. Seems difficult to accomplish without knowing the whole ins and outs of our system, and how highcharts are inserted.

Method 2 - (Almost) Pure CSS

Pure CSS is the more preferred approach but is not without limitations. Width changes are often not smooth, and any element that is resized through an animation won’t trigger a high charts redraw, so sizing (or resizing) the chart elements becomes difficult (at least doing so in a smooth manner)

Fades and slide in/outs likely give the best result since items live in a section off-screen and won’t be subject to size changes and potential redraws in the case of charts. In any event, javascript is needed to toggle the animation class on elements.

In the slide in example used here, I’m using a bootstrap modal but anchoring to the bottom of the viewport so it’s more of a tray. The styles have been tweaked to accommodate this.
